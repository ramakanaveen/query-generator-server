QConnect

Natural Language to KDB/Q Query Generation

Bridging the gap between questions and queries

The Query Challenge

  • 🕒
    Time Drain: Traders / Sales / Tech / Quants spend significant time crafting and debugging KDB/Q syntax
  • 🔄
    Context Switching: Disrupts analytical flow when switching between analysis and query writing
  • 🧠
    Knowledge Gap: Between "what data I need" and "how to express it in KDB/Q"
"Analysts typically spend 30-40% of their time just writing and debugging database queries"

The Cost of Complexity

  • 📈
    Steep Learning Curve: KDB / Q is complex. New team members take weeks to become productive with KDB/Q
  • ⚠️
    Error-Prone: Manual query writing introduces bugs and inconsistencies
  • 💰
    Opportunity Cost: Time spent on syntax could be used for actual analysis

Introducing QConnect

A FastAPI application that generates database queries from natural language using LLMs.

Key Features

  • 💬
    Natural Language to Database Query: Convert English questions to KDB/Q queries
  • 🤖
    Multi-LLM Support: Use either Google's Gemini or Anthropic's Claude models
  • 🔍
    Vector Search: Find relevant schemas using semantic similarity
  • 📊
    Schema Management: Upload and manage database schemas via API
  • 💾
    Conversation Context: Maintain context for follow-up questions

Intelligent Schema Matching

  • 📊

  • 🔍
    Semantic Search: Queries are matched to the most relevant tables using vector similarity
  • 🎯
    Configurable Threshold: Ensures quality matches (currently set at 0.65 similarity)

def get_model(self, model_name="gemini"):
    """
    Get a language model by name.
    
    Args:
        model_name: Name of the model to use ("gemini" or "claude")

        A LangChain chat model instance
    """
    if model_name not in self.models:

Schema Awareness with @Directives

Directives allow users to specify which schema to use:

@SPOT for spot market trading data
@STIRT for short-term interest rate data
@FXRates for foreign exchange Rates market data
@DSP for DSP strategy pricing and analytics
@FXFUSIONALGO for Algo pricing tables

Example:

"show me the EURUSD @STIRT Market Price for today"
def extract_directives(query):
    """Extract directives from the query text."""
    directives = []
    words = query.split()
    for word in words:
        if word.startswith('@'):
            directives.append(word.strip('@.,?!'))
    return directives

The Power of Context

QConnect maintains conversation context, allowing natural follow-ups:

User: "@STIRT Show me EUR Discount factors from yesterday"

Bot: [Returns query for EUR Discount factors from yesterday]

User: "Include GBP?"

Bot: [Understands this is a follow-up about GBP Discount factors from yesterday]

# Format conversation history for context
async def get_conversation_context(self, conversation_id, limit=5):
    """Get recent messages from a conversation for context."""
    if conversation_id not in self.conversations:
        return []
    
    messages = self.conversations[conversation_id]["messages"]
    return messages[-limit:] if messages else []
